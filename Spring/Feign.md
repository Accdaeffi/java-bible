# Что это вообще такое?
OpenFeign - декларативный инструмент привязки Java к HTTP-клиенту.
Работает на аннотациях, но лишь с текстовыми данными.

Если простым языком - это способ стучать к соседям по API как будто вызывая методы у себя. Максимально удобно и круто.

Со Spring работает неплохо - мы просто объявляем интерфейс, а Spring делает всё за нас.
Также нормально связывается с Eureka, CircuitBreaker и LoadBalancer.

# Как использовать?
1. Повесить @EnableFeignClients на конфиг.
2. Создать интерфейс, повесить на него @FeignClient(name = "имя"). Имя нужно как минимум для балансировки.
	2.1 Сюда же можно впихнуть балансировку - через поле "configuration" = Foo.class. При этом сам класс не должен быть аннотирован как @Congiguration, иначе оно будет для всех клиентов.
	2.2 Дефолтный конфиг можно настроить либо через EnableFeignClients, либо через файлы с параметрами.
3. В этом клиенте сделать старые-добрые методы-маппинги - с @RequestMapping, параметрами и возвращаемым объектом. Реализации не нужны, правда.
